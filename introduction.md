# Introduction 

Заметки из введения в "Learning Domain Driven Design" от Vladika Khononov. 

## Заметки

> Чтобы добиться успеха в программировании, нужно постоянно учиться — будь то изучение новых языков, освоение новых технологий или отслеживание свежих популярных фреймворков. Однако учить новый фреймворк для JavaScript каждую неделю — это ещё не самое сложное в нашей работе. Гораздо более серьёзная задача — разобраться в новых бизнес-доменах. 

Вкратце, суть из введения состоит в том, что DDD помогает решать бизнес задачи, тем спасая бизнес от software crisis. 

По сути, DDD можно расшифровать, как Business Domain Driven Software Design. 

DDD можно разделить на две части: стратегическую и тактическую.

Тактическая часть DDD используется для анализа бизнес-доменов, единого понимания бизнеса между всеми заинтересованными сторонами. 

Тактическая часть - позволяет писать код, чтобы он отражал бизнес-домен, соответствовал его целям и говорил на языке бизнеса.
